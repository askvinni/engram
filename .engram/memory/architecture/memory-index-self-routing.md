After issue #29, engram's memory architecture changed from flat category files (with CLAUDE.md listing each one) to individual per-topic files with YAML frontmatter. CLAUDE.md now holds exactly one @path reference — to `.engram/memory/index.md` — instead of N references to category files. The index is auto-generated by `rebuild_index()` in `src/memory.rs` and maps each topic file path to its `title` and `read_when` conditions. Agents read the index first, then selectively load only the files whose read_when conditions match their current task, keeping per-task token usage low. The consequence is that any new topic file written by `write_topic_file()` must trigger a `rebuild_index()` call to become discoverable; skipping it silently orphans the file. See src/memory.rs:write_topic_file and src/memory.rs:rebuild_index.
